// Activation

Setup in 5 steps.

  1. Download the NTS_Partner_Receiver.ex5 file using the button above. Save it somewhere you'll remember.
  2. Open MT5 → press Ctrl+Shift+D to open the Data Folder. Navigate to MQL5/Experts/ and drop the .ex5 file in. Close the folder.
  3. Restart MT5 (or right-click Expert Advisors in the Navigator → Refresh). The EA will appear under Expert Advisors as NTS_Partner_Receiver.
  4. Enable WebRequest for Firebase. In MT5: Tools → Options → Expert Advisors. Check Allow WebRequest for listed URL and add:
    https://[YOUR_PROJECT].firebaseio.com
    https://firestore.googleapis.com
    Your provider will tell you the exact project URL, or it's printed in the EA's first-run log.
  5. Attach to any chart (symbol doesn't matter — the EA subscribes to signals globally). In the EA inputs:
    • Paste the SUB- code your provider gave you.
    • Paste your Provider ID (format: NAV-XXXXX).
    • Pick your risk mode: Proportional, Fixed lot, or Mirror.
    Click OK, enable AutoTrading, and watch the EA comment line turn green when it's connected.
// Troubleshooting

Something not working?

"License invalid" error ?
(1) Double-check the code starts with SUB- and has no spaces. (2) Make sure your provider hasn't revoked it (ask them). (3) If this is the first activation on a new account, it's binding to this MT5 account number — once bound, it won't work on a different account until the provider reissues. (4) Check WebRequest URLs are enabled (see Step 4 above).
Signals come in but no trades open ?
Check (1) AutoTrading is enabled (green button top toolbar), (2) your broker supports the symbol the provider is trading (the resolver tries suffixes automatically but can fail on obscure brokers — set a manual map in the EA inputs), (3) your account has enough free margin, (4) the EA's "Allowed Symbols" filter isn't blocking it.
EA keeps reconnecting ?
Usually a VPS / WebRequest issue. Verify in Tools → Options → Expert Advisors that the Firebase URL is listed exactly (no trailing slash, no typo). Test by opening the URL in your VPS's browser — it should respond. Some Windows Server firewalls block outbound HTTPS; whitelist terminal64.exe.
My broker's symbol name is different ?
The EA's 4-layer resolver tries: exact match → common suffixes (m, .pro, #, .raw, etc.) → alias list → manual map. If you need to force a pair, edit the Symbol Map input: EURUSD:EURUSDm,XAUUSD:GOLD.
Can I use this on a prop-firm account? ?
Yes, but review your prop firm's EA rules first. Use the risk-cap, fixed-lot override, and symbol/magic skip filters. Many subscribers pass FTMO and MFF challenges using this EA — your mileage depends on your provider's style.
