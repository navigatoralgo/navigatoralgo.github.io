// Quick start

Five steps to your first signal.

  1. Install from MQL5 Market. If you haven't already, buy or download the demo from the MQL5 product page. After install, the EA shows up under Navigator → Expert Advisors → Trade Copier Ultimate.
  2. Allow WebRequest URLs in MT5. Open Tools → Options → Expert Advisors. Tick Allow WebRequest for listed URL and add both of these (one per line, no trailing slash):
    http://127.0.0.1
    https://api.telegram.org
    Without this, Telegram polling and the Bridge app cannot reach the EA. http://127.0.0.1 is only used for the local Bridge — nothing leaves your PC on that URL.
  3. Attach the EA to any chart. Symbol doesn't matter — the EA executes on the symbols inside each signal. Make sure AutoTrading is on (green button in the toolbar) and Allow Algo Trading is checked in the EA dialog.
  4. Pick a mode in the on-chart panel. Open the panel, go to the Source tab, and turn on exactly one of:
    • Telegram Bot API — paste your Bot Token and Chat ID. The bot must be admin in the channel/group.
    • Bridge Mode — set the Bridge Port (default 5555) and run the Navigator Command Center app on the same PC.
    • Discord Mode — paste your Discord Webhook URL.
    • Local Copier — choose MASTER on the source MT5 and SLAVE on the target MT5, with the same CopierFileName.
  5. Test, then ARM. Click Test Connection in the panel — you should see ONLINE for the mode you enabled. The EA starts in DISARMED state (no trades fire). Click ARM when you're ready. The header pill turns green and the next valid signal will execute.
Two safety toggles you should turn ON in the panel: Duplicate Filter (blocks the same signal arriving from two sources) and Entry Armour (rejects messages without a clear direction + symbol + price). Both ship OFF so they don't surprise advanced users running stress tests, but new buyers should turn them on once and forget about them.

Navigator Command Center · Bridge v7.41

The Bridge is a small Windows app that runs on the same PC as your MT5. It logs into your Telegram account (read-only via MTProto) so it can read private channels the Bot API can't reach, then routes per-account groups to the right MT5 terminal. Trade Copier Ultimate connects to it on http://127.0.0.1:<port> — nothing ever leaves your machine.

You don't need the Bridge for most setups. Bot API mode is enough for any Telegram channel where you can make your bot an admin (most public/group channels), and Discord mode handles webhooks directly. The Bridge is the answer only if your provider broadcasts in a private Telegram channel you can't bot-admin, or if you want one PC to route signals from multiple Telegram accounts to multiple MT5 terminals at once.

What your provider can send.

Signal typeExampleWhat it does

// Troubleshooting

Something not working?

Bridge says BLOCKED or Telegram won't connect
99% of the time this is the WebRequest checkbox in MT5. Open Tools → Options → Expert Advisors and confirm Allow WebRequest for listed URL is ticked, with both http://127.0.0.1 and https://api.telegram.org on separate lines (no trailing slash). Click OK, then click Test Connection in the EA panel again. If it still fails, your VPS or Windows firewall is blocking outbound HTTP from terminal64.exe — whitelist it.
Telegram bot is connected but no signals arrive
The bot must be an admin in the channel or group you're trying to read. Bot API mode cannot read messages otherwise. To check: open Telegram, go to the channel, tap the title → Administrators → confirm your bot is listed. If it's a private channel where you can't add bots, you need Bridge Mode (Navigator Command Center) instead. Also verify your Chat ID (negative number for groups, positive for DMs) — use @userinfobot to read it.
Signals come in but no trade opens
Walk down this list: (1) AutoTrading button green in the toolbar? (2) EA panel header pill says ARMED, not DISARMED? (3) Last Filtered on the Home tab — does it show a recent reject reason? (4) the broker actually has the symbol — check the Market Watch. (5) account has free margin to open the lot you've configured. The EA's diagnostic log (System tab → Diagnostic Log = ON) shows exactly which step rejected the signal.
The same trade opens twice
Two causes: (a) you have two sources turned on (e.g. Bot API and Bridge) and the same channel arrives in both — turn ON Filter → Duplicate Filter; (b) your provider re-sends the signal with slightly different formatting and the dedupe hash doesn't match — combine with a 3-minute Cooldown and Modify SL/TP If Position Exists = ON so a repeat signal updates instead of stacking.
Wrong lot size
Check the Lots tab. Lot Mode = Fixed with FixedLotSize = 0.10 always sends 0.10 — no auto-scaling. Risk % needs an SL on the signal (or a sane DefaultSLPoints fallback). Always verify MaxLotSize and Over Max Action: TRIM caps the lot, SKIP rejects the signal. On crypto/index symbols, the broker's contract size matters — test on demo first.
Pending order never fills
Either price didn't reach the entry (most common), or your broker has stop-level / freeze-level rules that rejected the order — the MT5 Experts log will say so. Pending Expiry deletes pending orders after the configured hours; if it's set too short, valid setups get cancelled before they fire. Check EnablePendingExpiry + PendingExpiryHours.
Broker symbol is named differently (e.g. BTCUSDm, XAUUSD.cash)
The universal symbol mapper handles common suffixes (m, .pro, #, .raw, .cash) automatically. If your broker uses something exotic, fill in Filter → Custom Symbol Mappings with a comma-separated list like EURUSD:EURUSDm,XAUUSD:GOLD. The first value is the signal's symbol, the second is your broker's.
Multi-TP split is missing a leg
Check that the signal actually has TP1 + TP2 + TP3 (or three sequential TP lines) and that SigTP → Multi-TP Split is ON. Verify Lot Distribution total ≤ 100% — if it's higher, the EA caps the last leg. The Max Signal TPs setting limits extraction (1, 2, or 3); if it's 1, you'll only get one trade no matter how many TPs the signal has.
Panel disappears or settings reset after restart
Panel state is auto-saved in MT5 GlobalVariables + a config file in MQL5/Files/Common/. If F7 inputs disagree with the saved panel state, the EA prefers the saved panel. To reset everything clean: remove the EA, run F3 → Global Variables, delete every variable starting with TCU_, then re-attach. If the panel visually disappears (no settings lost), drag the chart edge to redraw — known cosmetic glitch on rare DPI configurations.
EA freezes / lots of CPU on tick
Make sure you're on Hedging or Netting account type that matches your test plan — Netting accounts behave differently for multi-position ops. Lower BridgePollMs (default 1000 ms) and Bot Poll sec if you've set them too aggressive. Disable Diagnostic Log after setup is confirmed working — chatty logs slow things down on a busy chart. If you see runaway log spam, send the relevant lines to support and we'll trace it.
Can I run it on a prop-firm account?
Yes — turn ON Prop Firm Mode in the panel. It strips trade comments (some prop firms reject EA-tagged trades), enforces Require SL, and lets you cap daily loss in % or $. Combine with Auto SL fallback if your provider sometimes sends entries without SL. Always verify the prop firm allows EAs at all and that no DLLs are required (TCU is pure MQL5 — no DLLs).
Where are my settings / bot tokens stored?
Numeric and toggle settings live in MT5 GlobalVariables. String settings (bot tokens, webhooks, custom mappings) are saved as plain text in MQL5/Files/Common/TCU_*.cfg on your PC. Don't share that file — it contains your Telegram bot token, which gives full bot control to anyone who has it. If you ever post a settings export for support, redact the token line first.

What this EA does not do.

Trade Copier Ultimate is an execution utility. It does not generate trading signals or analysis. It does not guarantee profit or manage drawdown for you — that's your job and the job of the source you're copying. It does not include copy protection that limits your broker, account type, or number of charts.

Always test on a demo account before going live. Behaviour differs between Hedging and Netting accounts; verify your account type and re-run your edge cases. Past performance of any signal provider does not guarantee future results.

Demo versions from MQL5 run only inside the Strategy Tester. For end-to-end live testing on a demo broker account, use the paid version on a demo MT5 login.
